# ── Raw Planck source data  (Assets/CMB/Data/) ─────────────────────────────────

Assets/CMB/Data/COM_CMB_IQU-smica_2048_R3.00_full.fits
  type:   FITS
  size:   ~1.87 GB
  source: Planck 2018 CMB maps — SMICA IQU full-sky temperature + polarisation
  url:    https://pla.esac.esa.int/pla/#maps
  notes:  Nside=2048, ring ordering. Used by generate_particle_buffer.py to seed
          particle positions and bake HeegnerPower/VoidPressure/SolitonDensity.

Assets/CMB/Data/COM_PowerSpect_CMB-TT-full_R3.01.txt
  type:   plaintext (committed — 167 KB, safe to track)
  size:   167 KB
  source: Planck 2018 TT power spectrum
  url:    https://pla.esac.esa.int/pla/#powerspectra
  notes:  Columns: ell, D_ell, sigma. Used for acoustic peak weighting in
          Heegner node selection.

Assets/CMB/Data/Lensing/planck_lensing_kappa_MV.fits
  type:   FITS
  size:   ~160 MB
  source: Planck 2018 lensing — minimum-variance convergence κ map
  url:    https://pla.esac.esa.int/pla/#maps  (lensing section)
  notes:  Healpix Nside=2048. Feeds K_curvature.npz generation pipeline.

Assets/CMB/Data/Lensing/planck_lensing_mask.fits.gz
  type:   FITS (gzipped)
  size:   ~424 KB
  source: Planck 2018 lensing analysis mask
  url:    same as above
  notes:  Applied during κ map processing to exclude foreground-contaminated pixels.

Assets/CMB/Data/MilkyWay/
  type:   directory (contents TBD — to be catalogued)
  notes:  Milky Way foreground / dust maps. Placeholder for masking pipeline.


# ── Computed pipeline outputs  (Assets/CMB/Data/) ─────────────────────────────
# Generated by generate_particle_buffer.py from the raw Planck data above.
# Regenerate with:  python generate_particle_buffer.py --config field_scalars.json

Assets/CMB/Data/alm_by_class.npz
  type:   NumPy archive
  size:   ~1 GB
  keys:   alm_soliton, alm_void, alm_heegner  (complex128 arrays, ell_max=2048)
  notes:  Spherical harmonic coefficients decomposed by particle class.
          Used to reconstruct class-specific sky maps for visual validation.

Assets/CMB/Data/K_curvature.npz
  type:   NumPy archive
  size:   ~55 MB
  keys:   kappa, kappa_smoothed, pixel_indices
  notes:  Lensing curvature κ evaluated at particle positions.
          Drives Kappa field in ParticleRecord (offset 13).

Assets/CMB/Data/nodes.npz
  type:   NumPy archive
  size:   ~2 MB
  keys:   positions (float32 xyz), weights, class_ids
  notes:  Heegner/anchor node positions on unit sphere in Galactic Cartesian.
          Source for anchor_positions.bin (see StreamingAssets below).

Assets/CMB/Data/frame_vectors.json
  type:   JSON  (committed — 4 KB, safe to track)
  notes:  ZaTaOa orthonormal frame vectors + GalToUnity rotation matrix.
          Consumed by HolographicParticleLayer.cs SetGalToUnity() at startup.

Assets/CMB/Data/field_scalars.json
  type:   JSON  (committed — safe to track)
  notes:  Runtime field scalar config: parity_asymmetry, heegner_fraction,
          void_fraction, soliton_fraction, xi_baseline.
          See FieldScalars struct in HolographicParticleLayer.cs.

Assets/CMB/Data/K_stats.json
  type:   JSON  (committed — safe to track)
  notes:  Summary statistics for the κ curvature field (mean, std, percentiles).
          Used to normalise Kappa values into [0,1] for the ParticleRecord.


# ── Runtime buffers  (Assets/StreamingAssets/CMB/) ────────────────────────────
# Loaded at runtime by HolographicParticleLayer.cs via Application.streamingAssetsPath.
# Must be present on device before first launch (Quest 2: copy to persistentDataPath).

Assets/StreamingAssets/CMB/particle_buffer.bin
  type:   raw binary, stride=96 bytes (ParticleRecord struct)
  size:   depends on particle count (N * 96 bytes)
  struct: float3 Position | float3 Velocity | float4 Colour |
          float HeegnerPower | float VoidPressure | float SolitonDensity |
          float Kappa | int32 ParticleClass | float ParityWeight | float[8] padding
  notes:  Primary GPU buffer. Written by generate_particle_buffer.py.
          Read by hpl_viz_C.py for Python preview renders.

Assets/StreamingAssets/CMB/epoch_frame.json
  type:   JSON  (committed — safe to track)
  notes:  Per-epoch ZaTaOa frame snapshot used to pin GalToUnity at t=0.
          See Docs/holographic-projection.md "Open Design Questions".

Assets/StreamingAssets/CMB/anchor_positions.bin
  type:   raw binary, stride=12 bytes (float3 xyz on unit sphere)
  size:   _anchorCount * 12 bytes
  status: NOT YET GENERATED — LoadAnchorBuffer() falls back to NGP (0,0,1).
  notes:  Derived from nodes.npz positions field.
          Generate with:  python generate_particle_buffer.py --write-anchors
