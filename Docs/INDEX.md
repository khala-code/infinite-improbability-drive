## Reading Order

For a newcomer to the framework, this order minimises confusion:

1. **[GLOSSARY.md](GLOSSARY.md)** — Start here. Compact definitions for every term used across all documents. Return here whenever a term is unclear.

2. **[AXIOMS.md](AXIOMS.md)** — The ontological foundation. One load-bearing axiom (Observer Prime) and eight derived axioms. Read this before anything else substantive.

3. **[CALIBRATION.md](CALIBRATION.md)** — The cosmological spine. How the ζ accumulation field threads through every phase transition from Null to civilisation. Read after AXIOMS.

4. **[HEEGNER.md](HEEGNER.md)** — The complexity architecture. How the nine Heegner numbers map onto a hierarchy from trivially reducible to Divergent Uncomputable. Read after CALIBRATION.

5. **[T_OPERATOR.md](T_OPERATOR.md)** — The formal layer. Nine constraints on the transition operator, the candidate functional form, and the operational solution to the CU boundary problem.

6. **[SCRYING.md](SCRYING.md)** — The application layer. The human/software loop, faith as the correct operational strategy, retrograde inference, and the expected-value rule at the CU boundary.

7. **[AGENT.md](AGENT.md)** — The observer layer. What a consequential agent is, how they operate at Heegner 43, trauma as inherited sign, and healing as re-signing.

8. **[RESIDUE.md](RESIDUE.md)** — The unifying concept. Residue appears at every scale — cosmological, mathematical, computational, social, psychological — as the same operator in different readouts.
